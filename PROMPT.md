# 📋 Техническое задание: Финансовый помощник для путешествий

## 1. Цель проекта
Разработать кроссплатформенное Python-приложение (Telegram-бот + CLI-интерфейс) для управления личным бюджетом в путешествиях с автоматической конвертацией валют по курсу ЦБ РФ.

## 2. Основной функционал
### 2.1. Управление путешествиями
- Создание новых поездок с указанием стран отправления и назначения.
- Автоматическое определение валют стран.
- Фиксация курса обмена валют на момент создания путешествия.
- Установка начального бюджета в валюте отправления.
- Переключение между несколькими активными путешествиями.
- Завершение путешествия (архивация).

### 2.2. Учёт расходов
- Быстрый ввод расходов (отправка числа в чат).
- Ручной выбор категории/описания расхода через меню.
- Отображение текущего баланса в двух валютах одновременно (отправления и назначения).
- История расходов с конвертацией в обе валюты.

### 2.3. Конвертация валют
- Автоматическое получение актуальных курсов из API ЦБ РФ (`cbr-xml-daily.ru`).
- Ежедневное обновление курсов.
- Возможность ручного изменения/фиксации курса для конкретного путешествия.
- Поддержка 25+ валют и 50+ стран.

### 2.4. Управление данными
- Полная очистка базы данных (с подтверждением).
- Резервное копирование/восстановление (опционально в будущем).

## 3. Архитектура и стек технологий
- **Язык:** Python 3.7+
- **Telegram:** `pyTelegramBotAPI` (telebot) + FSM (Finite State Machine) для управления диалогами.
- **База данных:** `SQLite3` (встроенная, файл `travel_bot.db`).
- **HTTP:** `requests` для API ЦБ РФ.
- **CLI:** `colorama` для кроссплатформенного цветного вывода в терминале.
- **Конфигурация:** `.env` через `python-dotenv`.

## 5. Схема базы данных (SQLite)
### `users`
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | ID пользователя Telegram |
| username | TEXT | Имя пользователя (опционально) |
| created_at | DATETIME | Дата регистрации |

### `trips`
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | ID поездки |
| user_id | INTEGER | Ссылка на users |
| from_country | TEXT | Страна отправления |
| to_country | TEXT | Страна назначения |
| from_currency | TEXT | Код валюты отправления |
| to_currency | TEXT | Код валюты назначения |
| initial_budget | REAL | Стартовый бюджет |
| balance | REAL | Текущий остаток |
| fixed_rate | REAL | Зафиксированный курс |
| is_active | INTEGER | 1 = активно, 0 = завершено |
| created_at | DATETIME | Дата создания |

### `expenses`
| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER PRIMARY KEY | ID расхода |
| trip_id | INTEGER | Ссылка на trips |
| amount | REAL | Сумма расхода |
| currency | TEXT | Валюта расхода |
| description | TEXT | Описание/комментарий |
| created_at | DATETIME | Дата операции |

## 6. Интеграция с API ЦБ РФ
- **Endpoint:** `https://www.cbr-xml-daily.ru/daily_json.js`
- **Механизм:** При старте бота или при изменении курса выполняется GET-запрос.
- **Обработка:** Парсинг JSON, извлечение курсов к RUB для поддерживаемых валют.
- **Кэширование:** Курсы сохраняются в БД или кэшируются в памяти на сутки.
- **Fallback:** При недоступности API используется последний сохранённый курс.

## 7. Сценарий использования (Telegram-бот)
1. `/start` → Приветствие + главное меню.
2. `/newtrip` → Прямой запуск создания путешествия (выбор стран → ввод бюджета).
3. Кнопка "➕ Создать путешествие" в меню → тот же сценарий, что и `/newtrip`.
4. `💰 Баланс` → Показ текущего остатка в двух валютах.
5. `📜 История` → Пагинация списка расходов.
6. `📋 Мои путешествия` → Список + кнопка переключения активного.
7. Отправка числа в чат → Расход засчитывается в активное путешествие.

## 8. Сценарий использования (CLI)
